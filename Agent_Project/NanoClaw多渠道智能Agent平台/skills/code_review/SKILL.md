## 使用方法

当用户要求审查代码时，按以下步骤操作：

1. 使用 `read_file` 工具读取用户指定的代码文件
2. 仔细分析代码，关注以下方面：
   - **Bug**：逻辑错误、边界条件、空指针、类型错误
   - **安全隐患**：SQL 注入、路径穿越、硬编码密钥、不安全的反序列化
   - **代码风格**：命名规范、函数过长、重复代码
   - **性能问题**：不必要的循环、内存泄漏、大 O 复杂度过高
3. 按严重程度排序输出问题列表
4. 对每个问题给出具体的修复建议和代码示例

## 输出格式
🔴 严重问题
[文件名:行号] 问题描述
修复建议：...
🟡 一般问题
[文件名:行号] 问题描述
修复建议：...
🟢 改进建议
建议描述
改进方式：...

## 注意事项

- 不要只说"代码不好"，要给出具体的行号和修复代码
- 安全问题优先级最高，一定要首先指出
- 如果代码写得不错，也要明确说"没有发现严重问题"
- 对于风格问题，参考 PEP 8 规范
