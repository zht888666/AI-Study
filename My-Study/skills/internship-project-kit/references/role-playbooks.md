# Role Playbooks

Use these defaults when matching JD roles to project types and modifications.

## Backend

Prefer API systems with database, authentication, caching, async tasks, tests, and docs. Good modifications: add REST endpoint, replace SQLite with MySQL/PostgreSQL, add Redis cache, add integration tests, add OpenAPI docs.

## Frontend

Prefer dashboards, admin systems, ecommerce, monitoring views, or data visualization apps. Good modifications: add a page, improve state management, connect real API, add loading/error states, add component tests.

## Full Stack

Prefer small end-to-end apps. Good modifications: connect frontend to backend, add auth, add CRUD flow, add deployment docs, add tests.

## Mobile

Prefer Android/iOS apps with API integration. Good modifications: add screen, local storage, network error handling, login flow, testable module.

## Test Development

Prefer projects with API/UI flows and missing tests. Good modifications: add pytest/JUnit, API contract tests, Playwright/Cypress tests, CI checks, test report.

## Data Engineering

Prefer ETL, data pipeline, log processing, warehouse demo, or streaming projects. Good modifications: add data validation, incremental loading, scheduling, monitoring, quality report.

## Cloud Native / DevOps

Prefer Dockerized services, Kubernetes demos, monitoring stacks, CI/CD examples. Good modifications: Dockerfile, Compose, health check, GitHub/Gitee CI, Prometheus metrics.

## Security

Prefer auth, scanner, web security labs, audit tools. Good modifications: fix auth bug, add JWT/session security, add dependency scan, add threat model, add security tests.

## Systems

Prefer storage, scheduler, network, database, distributed systems, or performance-oriented projects. Good modifications: benchmark, profiling, lock/concurrency fix, cache, queue, memory optimization.

## AI / Algorithm

Prefer RAG, document QA, model inference API, recommendation, search, or experiment projects. Good modifications: add evaluation set, compare embeddings, add retrieval metrics, expose inference API, improve prompt pipeline.
