# Interview Pack Playbook

Generate material the user can defend honestly.

## STAR Resume Project

Structure:

- Situation: JD-related problem and project background.
- Task: candidate-owned goal.
- Action: concrete completed work, planned modifications, and tools used.
- Result: measured or observable result; if not measured, state what can be measured.

Keep bullets specific and avoid inflated ownership claims.

## Core Code Walkthrough

Cover:

- Main entrypoint.
- Request/page/task flow.
- Important modules.
- Data model and storage.
- Error handling.
- One tradeoff and one improvement.

## Interview Q&A

Include questions on:

- Why this project matches the JD.
- How it starts locally.
- Data/API/task flow.
- Bottlenecks.
- Failure modes.
- Security or reliability.
- What the candidate personally changed.
- What would be improved next.

## PPT Prompts

Generate slide prompts for:

- Project background.
- Architecture.
- Core workflow.
- Baseline run.
- Modification highlights.
- Result and reflection.

## Submission Checklist

Check:

- Resume bullet does not exaggerate.
- Project source is attributed.
- Run depth is stated.
- Screenshots or logs are ready when available.
- Talking points match actual code.
